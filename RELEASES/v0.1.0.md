# v0.1.0 — Safe local ops agent

Tagline: Safe local ops agent: patch→deploy→health→invalidate, all dry-run first.

## Highlights
- Safe patching: unified diff → ops(JSON) → apply with workspace confinement.
- Declarative recipes: `agent run-recipe --recipe agent.yml --dry-run --format=text|json`.
- Local-first: Node.js stdlib only, no network calls unless explicit (dry-run respected).
- Logs: `logs/YYYYMMDD-HHMM/{plan.json,run-recipe.log[,run.json]}` are saved per run.
- Secrets masking: `secret/token/authorization` are masked in plan/run outputs.

## What’s included
- Core
  - `src/core/{loader,runner}.js` — load recipe (JSON/YAML-light), plan, run.
  - `src/lib/{apply,patch,exec,ssh,log,mask}.js` — patch conversion, safe writes, logging, masking.
- Commands
  - `run-recipe`, `deploy`, `check`(=health), `revalidate`(=invalidate), `logs`, `fix`, `patch`, `ingest`, `serve`.
  - All accept `--dry-run`.
- Adapters (minimal)
  - `adapters/next#resolveRevalidate` resolves endpoint/secret/path/slug (priority: env > recipe > default).
  - `adapters/strapi` placeholder for future info injection.
- Examples
  - `examples/static/agent.yml` — deploy + healthcheck.
  - `examples/next/agent.yml` — deploy + revalidate(slug=/hello-strapi) + healthcheck.
  - `examples/strapi/agent.yml` — deploy + healthcheck.
- Docs
  - Updated `agent-cli/README.md` (Quick Start, safety, examples, links to SECURITY/CONTRIBUTING).
  - `SECURITY.md`, `CONTRIBUTING.md` (draft).
- CI
  - Node 18, unit tests, `run-recipe` dry-run for static/next; upload `logs/` artifact.

## Quick Start
```bash
node agent-cli/bin/agent help
node agent-cli/bin/agent run-recipe --recipe examples/static/agent.yml --dry-run --format=json
node agent-cli/bin/agent patch --input patch.diff --apply --dry-run
```

## Safety
1) Workspace confinement: writes only under CWD; `..` / absolute / symlink targets are refused.
2) Secrets masking: plan/run outputs mask `secret`, `token`, `authorization`.
3) Dry-run first: all actionful commands accept `--dry-run`.

## Known Limitations
- Rename is supported; file mode changes are not (by design in v0.1).
- Symlinks are not followed.
- Windows path semantics are not supported yet.

## Roadmap (v0.2+)
- Split adapters to packages (e.g. @ops-agent/adapter-next).
- `agent init` for interactive recipe bootstrap.
- `--parallel` for independent steps (v0.1 stays serial).
